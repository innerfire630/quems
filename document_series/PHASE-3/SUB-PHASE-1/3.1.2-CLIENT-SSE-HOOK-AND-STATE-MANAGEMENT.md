## 1. Document Purpose

This document delivers the **client-side consumer** of the SSE infrastructure built in 3.1.1. It introduces the **`useSSE()` React hook** at `src/hooks/use-sse.ts` — the single entry point that every client component (display board, audio system, officer dashboard, security screen) uses to subscribe to real-time events. The hook opens an `EventSource` connection, parses incoming events into the typed envelope, invokes a caller-provided callback for each event, and **reconnects automatically with exponential backoff** (1s, 2s, 4s, 8s, max 30s) when the connection drops. It also introduces the **`SseDebugPanel` component** and a **temporary SSE test page** at `src/app/(debug)/sse-test/page.tsx` that together form the verification surface — the agent opens the test page in a browser, triggers an event from the server side (via a debug script or a manual test of any Phase 2 endpoint), and confirms the event appears in the panel within a second.

When this document is complete:

- Any client component can call `useSSE('global', { onEvent: (envelope) => { ... } })` and receive typed events.
- Killing the network and restoring it triggers automatic reconnection with exponential backoff.
- The hook is generic over the event type filter — passing `{ filter: 'TICKET_CALLED' }` types the callback's `envelope.payload` correctly (no casting required).
- The hook cleans up on unmount (the `EventSource` is closed, the reconnection timer is cleared).
- The `SseEnvelope<T>` type and the `SseEventTypeMap` constant are exported from `src/types/sse.types.ts` as the type-level single source of truth for what events exist.
- The temporary `/sse-test` debug page demonstrates the hook working end-to-end with a real SSE connection.

This document does **not** implement the display board UI (owned by 3.2), the audio announcement system (owned by 3.3), the real `broadcastEvent()` body (owned by 3.1.3), or the SSE manager itself (owned by 3.1.1).

### Step 1: Extend `src/types/sse.types.ts` with Envelope, Type Map, and Connection Event Constants

**Goal:** Finalise the type-level contract for SSE events on the client side.

**Files affected:** `src/types/sse.types.ts` (modified).

**Specific instructions:**

- Open the existing `src/types/sse.types.ts` (created in 2.1.1, extended in 2.2.1, 2.3.1, 2.3.2, 2.3.3).
- **Do NOT modify the existing `SseEventPayload` discriminated union.** It is frozen in 3.1.
- **Add** the following:
  - **`SseEnvelope<T extends SseEventType>`** — a generic type:
    - `type: T`
    - `id: string`
    - `timestamp: string`
    - `payload: SseEventPayload[T]`
    - Document that `T` is the event type and `payload` is typed correctly via the discriminated union.
  - **`SseEventTypeMap`** — a constant of type `Record<SseEventType, SseEventPayload[SseEventType]>`. Implementation: a frozen object literal that maps each `SseEventType` to its payload type. **Important:** this is a TYPE-LEVEL mapping, not a runtime value. The agent uses `as const` and explicit field declarations to ensure TypeScript infers the correct types.
    - For each `SseEventType` value Phase 2 defined, declare a field whose value is `null as unknown as SseEventPayload[<type>]`. (The `null as unknown as` cast is the standard TypeScript pattern for type-only object literals.)
    - Document at the top of the constant that it is for type inference only — do not use the runtime values.
  - **`SSE_INTERNAL_EVENTS`** — a constant array of event types that are emitted by the SSE infrastructure itself but NOT via `broadcastEvent()`:
    - `'CONNECTED'` — sent by the route handler on connection establishment.
    - Future: `'HEARTBEAT_LOST'` (Phase 5 may add this for monitoring).
  - **`SseInternalEventType`** — a union type extracted from `SSE_INTERNAL_EVENTS` (currently just `'CONNECTED'`).
  - **`SseInternalEnvelope`** — the envelope shape for internal events (separate from `SseEnvelope<T>` because internal events do not have a payload in the discriminated union). Fields: `type: SseInternalEventType`, `id: string`, `timestamp: string`, `payload: { clientId: string, channel: string }` (for `CONNECTED`).
- Document that `SseEnvelope<T>` is the canonical shape consumers receive from `useSSE()`, and that `SseInternalEnvelope` is separate and only relevant for connection-state tracking.

### Step 3: Create the `useSSE()` Hook

**Goal:** Build the core client-side hook.

**Files affected:** `src/hooks/use-sse.ts` (new file).

**Specific instructions:**

Create `src/hooks/use-sse.ts` containing:

- **Imports:**
  - `useEffect`, `useRef`, `useState`, `useCallback` from `react`.
  - `SseEventType`, `SseEventPayload`, `SseEnvelope`, `SseInternalEnvelope`, `SseInternalEventType` from `@/types/sse.types`.
- **Type definitions (local):**
  - `type SseConnectionStatus = 'idle' | 'connecting' | 'open' | 'reconnecting' | 'closed'`.
  - `type SseReconnectSchedule = readonly number[]` — the backoff schedule in milliseconds: `[1000, 2000, 4000, 8000, 30000]` (1s, 2s, 4s, 8s, max 30s).
  - `interface UseSSEOptions<T extends SseEventType> { onEvent?: (envelope: SseEnvelope<T>) => void; onInternalEvent?: (envelope: SseInternalEnvelope) => void; filter?: T | readonly T[]; }`.
  - `interface UseSSEResult { status: SseConnectionStatus; reconnectAttempts: number; lastEventAt: Date | null; totalEventsReceived: number; }`.
- **Constants:**
  - `MAX_RECONNECT_DELAY_MS = 30000`.
  - `RECONNECT_SCHEDULE: readonly number[] = [1000, 2000, 4000, 8000]` (the schedule before clamping at the max).
- **Function `useSSE<T extends SseEventType = SseEventType>(channel: string, options?: UseSSEOptions<T>): UseSSEResult`**:
  - **State (via `useState`):**
    - `status: SseConnectionStatus` — starts as `'idle'`.
    - `reconnectAttempts: number` — starts at `0`.
    - `lastEventAt: Date | null` — starts at `null`.
    - `totalEventsReceived: number` — starts at `0`.
  - **Refs (via `useRef`):**
    - `eventSourceRef: EventSource | null` — the current `EventSource` instance. (Held in a ref so the cleanup function and the reconnection logic can access it without re-creating on every render.)
    - `reconnectTimerRef: NodeJS.Timeout | null` — the pending reconnection timer.
    - `isUnmountedRef: boolean` — flag set to `true` in the cleanup function. Used to prevent state updates after unmount (React warning: "Can't perform a state update on an unmounted component").
    - `reconnectAttemptsRef: number` — mirror of the `reconnectAttempts` state, held in a ref so the reconnection logic doesn't trigger re-renders.
  - **`useEffect` setup:**
    - Runs on mount and when `channel` or `filter` changes.
    - **Cleanup (returned from the effect):**
      - Set `isUnmountedRef.current = true`.
      - If `reconnectTimerRef.current !== null`, clear it.
      - If `eventSourceRef.current !== null`, close it (`eventSourceRef.current.close()`).
    - **Setup body:**
      - Set `isUnmountedRef.current = false`.
      - Set `status` to `'connecting'`.
      - Compute the URL: `${window.location.origin}/api/sse/${channel.replace(/:/g, '-')}` (the dash-to-colon conversion from 3.1.1's chosen path approach). Note: in the browser, `window.location.origin` is always defined. For SSR safety, check `typeof window !== 'undefined'` and skip if not in the browser (the hook is "use client" anyway, but defensive).
      - Create `new EventSource(url)`. Assign to `eventSourceRef.current`.
      - **Handler `eventSource.onopen`:**
        - If `isUnmountedRef.current`, return.
        - Set `status` to `'open'`, `reconnectAttempts` to `0`, `reconnectAttemptsRef.current` to `0`.
      - **Handler `eventSource.onmessage`:**
        - Parse the event's `data` field as JSON.
        - If parsing fails, log via `console.warn` and skip the event.
        - If the parsed `type` is in `SSE_INTERNAL_EVENTS` (`'CONNECTED'`):
          - Build an `SseInternalEnvelope`.
          - Invoke `options?.onInternalEvent?.(envelope)` if provided.
          - Skip the regular event handling path.
        - Else (regular event):
          - Apply the `filter` if provided. If `filter` is a single event type, check `parsed.type === filter`. If `filter` is an array, check `filter.includes(parsed.type)`. If the filter does not match, skip.
          - Update `lastEventAt` and `totalEventsReceived`.
          - Invoke `options?.onEvent?.(envelope as SseEnvelope<T>)` if provided. The `as SseEnvelope<T>` cast is safe because the filter has narrowed the type — the TypeScript inference flows from the `filter` parameter.
      - **Handler `eventSource.onerror`:**
        - **Important:** the hook does NOT use the built-in `EventSource` reconnection. It implements its own.
        - If `isUnmountedRef.current`, return.
        - Set `status` to `'reconnecting'`.
        - Close the current `eventSource` (`eventSourceRef.current?.close()`).
        - Compute the next reconnection delay:
          - `const delay = RECONNECT_SCHEDULE[Math.min(reconnectAttemptsRef.current, RECONNECT_SCHEDULE.length - 1)]` or `MAX_RECONNECT_DELAY_MS` (whichever is smaller — since the last schedule value is 8000ms, this clamps it).
          - Increment `reconnectAttemptsRef.current` by 1.
          - Update `reconnectAttempts` state to match.
        - Set `reconnectTimerRef.current = setTimeout(() => { ... re-attempt setup ... }, delay)`.
        - The re-attempt setup creates a new `EventSource` with the same URL, re-wires the handlers (which reference the latest refs).
      - **Return cleanup** (as above).
  - **`useEffect` dependencies:** `[channel, JSON.stringify(filter)]` — the filter array is stringified so a new array with the same contents does not trigger re-subscription.
- **Return value:**
  - `{ status, reconnectAttempts, lastEventAt, totalEventsReceived }`.
- **Notes for the agent:**
  - The hook MUST be called from inside a React component or another hook (React rules of hooks).
  - The hook MUST NOT be called inside loops or conditions.
  - Multiple subscriptions in a single component require multiple `useSSE()` calls.
  - The `filter` is applied AFTER the JSON parse, so the cost is minimal.

- **State:**
  - `eventLog: Array<{ receivedAt: Date; envelope: SseEnvelope<SseEventType> | SseInternalEnvelope }>` — bounded at 20 entries (oldest dropped on overflow).
- **Hook usage:**
  - `const { status, reconnectAttempts, lastEventAt, totalEventsReceived } = useSSE<SseEventType>(channel, { onEvent: (envelope) => { ... append to eventLog ... }, onInternalEvent: (envelope) => { ... append to eventLog ... } })`.
- **Render:**
  - A `Card` containing:
    - **Header section:**
      - Title: "SSE Debug Panel".
      - Channel display: `Channel: <channel>`.
      - Status badge: `<Badge>` colored by status (`open` = green, `connecting` = yellow, `reconnecting` = orange, `closed` = red, `idle` = gray).
      - Reconnect attempts display: `Reconnect attempts: <N>` (only show if > 0).
      - Total events received: `Total events: <N>`.
      - Last event time: `Last event: <relative time>` (e.g., "2 seconds ago" via `Intl.RelativeTimeFormat`).
    - **Event log section:**
      - A scrollable list (max-height, overflow-y: auto) showing each event:
        - Timestamp (HH:MM:SS.mmm).
        - Event type (as a badge).
        - Payload (collapsed by default, expandable on click).
        - Event ID.
      - The list is ordered most-recent-first.
  - The component re-renders on every event arrival because the parent state (`eventLog`) updates.
- **Auto-scroll behaviour:**
  - The event log container has `overflow-anchor: auto` (or a similar CSS pattern) so new entries push older ones off-screen but the user can scroll up to see them.
- **Styling:**
  - Uses shadcn UI components for consistent visual style with the rest of the admin dashboard.
  - The status badge colors map to the system's standard color tokens: success (green) for `open`, warning (yellow) for `connecting`/`reconnecting`, destructive (red) for `closed`.

## 8. Common Pitfalls

1. **Using `EventSource`'s built-in reconnection.** The built-in reconnection has no backoff control — it just retries at the default interval. Fix: implement custom reconnection in `onerror`.
2. **Forgetting cleanup on unmount.** Without it, the `EventSource` and the reconnection timer leak. Symptom: connections accumulate, browser memory grows. Fix: return a cleanup function from `useEffect` that closes the `EventSource` and clears the timer.
3. **Updating state after unmount.** React warns about state updates on unmounted components. Fix: set `isUnmountedRef.current = true` in cleanup and check it before any state update.
4. **Re-subscribing on every render.** If `filter` is an inline array (`{ filter: ['TICKET_CALLED'] }`), the array's identity changes every render, causing infinite re-subscription. Fix: stringify the filter for the dependency array (`[channel, JSON.stringify(filter)]`), or accept that callers pass stable references.
5. **Logging payload contents in the debug panel.** The panel shows payloads — this is intentional for debugging. But if the agent logs payloads to `console.log`, PII leaks into the browser console. Fix: log only `type` and `id` to console, never the full payload.
6. **Forgetting to handle `CONNECTED` separately.** The `CONNECTED` event is NOT in the `SseEventPayload` union. If the hook tries to invoke `onEvent` with it, the type system rejects the call. Fix: the hook routes `CONNECTED` through `onInternalEvent` instead of `onEvent`.
7. **Not closing the `EventSource` before reconnecting.** Two open connections to the same endpoint on reconnect leaks one. Fix: always `eventSource.close()` before creating a new one.
8. **Putting the hook in a server component.** `EventSource` and `window` are browser-only. The hook is "use client", but the agent must verify the consuming component is also a client component.
9. **Allowing production access to debug pages.** The `(debug)/layout.tsx` `notFound()` guard is critical — without it, anyone in production can see internal SSE state. Fix: confirm the guard works in `yarn build` output.
10. **Forgetting the dash-to-colon conversion.** If 3.1.1 chose Option A (dash separator in the URL), the hook must convert dashes back to colons before opening the `EventSource`. Otherwise the URL is wrong and the route rejects the channel.

## 9. AI Agent Guidance

This section is specifically for AI agents picking up this task plan document.

### 9.1 Critical Implementation Notes

- **The hook MUST be a "use client" hook.** It uses `EventSource`, `window`, and `setTimeout` — all browser-only APIs. The agent confirms the file has a `"use client"` directive at the top.
- **The hook MUST use its own reconnection logic.** The built-in `EventSource` reconnection has no backoff control. The agent implements exponential backoff manually in the `onerror` handler.
- **The hook MUST clean up on unmount.** This is non-negotiable. Without cleanup, connections leak and the browser accumulates `EventSource` instances.
- **The hook MUST handle the `CONNECTED` internal event separately.** It is not part of the `SseEventPayload` union. Routing it through `onInternalEvent` keeps the type system honest.
- **The `filter` parameter MUST use stable references for the dependency array.** Passing an inline array causes infinite re-subscription. The agent accepts this is the consumer's responsibility but uses `JSON.stringify` for the dependency comparison to be defensive.
- **The debug page MUST NOT be accessible in production.** The `(debug)/layout.tsx` `notFound()` guard is the enforcement mechanism.
- **The `SseEventPayload` discriminated union is FROZEN.** Adding new variants here is out of scope. Future sub-phases that introduce new event types (Phase 4.1 for FCM-related events, etc.) will add them explicitly.
- **The type-level inference for `envelope.payload` MUST flow from `filter` to the callback parameter.** The `SseEnvelope<T>` generic + the filter narrowing is what enables this. If the agent casts manually (`as any`), the type safety is lost.

### 9.2 What the Agent Should NOT Do

- Do not use `EventSource`'s built-in reconnection (no backoff control).
- Do not add new `SseEventType` variants.
- Do not put the hook in a server component.
- Do not log payload contents to `console.log`.
- Do not skip the cleanup function in `useEffect`.
- Do not allow production access to debug pages.

### 9.3 What to Do When Stuck

- If the hook doesn't reconnect after a network drop, the `onerror` handler may not be firing. Verify with a `console.debug` inside `onerror`.
